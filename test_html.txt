            <h1>
              
              Phase 1: Identity and Access Management (IAM)
            </h1>
          

        



	<h1 id="identity-and-access-management-iam">Identity and Access Management (IAM)</h1>
<p>Setting up Identity and Access Management (IAM) is the first and most crucial foundation to ensure that services within the <strong>AuraAcademic</strong> architecture can securely communicate with each other, and the CI/CD system (GitHub Actions) has sufficient permissions to update the source code.</p>
<p>In this step, we will create an <strong>IAM User</strong> for GitHub Actions and ensure the <strong>ECS Task Execution Role</strong> exists.</p>
<hr>
<h3 id="step-1-create-iam-user-for-github-actions">Step 1: Create IAM User for GitHub Actions</h3>
<p>GitHub requires a credentialed account to automate the process of pushing Docker Images to ECR and commanding ECS to update the application.</p>
<ol>
<li>Log in to the <strong>AWS Management Console</strong>, search for <code>IAM</code> and select <strong>IAM</strong>.</li>
<li>On the left menu, select <strong>Users</strong>, then click <strong>Create user</strong>.</li>
<li>In the <strong>User details</strong> section:
<ul>
<li>User name: enter <code>github-actions-deploy</code>.</li>
<li>Click <strong>Next</strong>.</li>
</ul>
</li>
<li>In the <strong>Set permissions</strong> section:
<ul>
<li>Choose <strong>Attach policies directly</strong>.</li>
<li>Search for and check the following policies:
<ol>
<li><code>AmazonEC2ContainerRegistryFullAccess</code> (Permission to interact with Docker Image repository - ECR)</li>
<li><code>AmazonECS_FullAccess</code> (Permission to update applications on ECS)</li>
<li><code>AmazonS3FullAccess</code> (Permission to upload Frontend files to S3)</li>
<li><code>CloudFrontFullAccess</code> (Permission to invalidate CDN cache after uploading new frontend)</li>
</ol>
</li>
</ul>
</li>
<li>Click <strong>Next</strong> to the Review page, then click <strong>Create user</strong>.</li>
</ol>
<p><img src="../../images/5-Workshop/5.1-IAM-Security/1-create-user.png" alt="Create IAM User for GitHub Actions"></p>
<hr>
<h3 id="step-2-create-access-key-credentials">Step 2: Create Access Key (Credentials)</h3>
<p>After successfully creating the User, we need to generate access keys for GitHub.</p>
<ol>
<li>Click on the user <code>github-actions-deploy</code> you just created.</li>
<li>Switch to the <strong>Security credentials</strong> tab.</li>
<li>Scroll down to the <strong>Access keys</strong> section, click <strong>Create access key</strong>.</li>
<li>Select <strong>Third-party service</strong>, check the confirmation box, then click <strong>Next</strong>.</li>
<li>Click <strong>Create access key</strong>.</li>
<li><strong>IMPORTANT:</strong> The screen will display the <code>Access key ID</code> and <code>Secret access key</code>.
<ul>
<li>Open Notepad/Text editor and copy these 2 keys to save them.</li>
<li><em>The Secret access key cannot be viewed again after you click Done!</em></li>
</ul>
</li>
</ol>
<p><img src="../../images/5-Workshop/5.1-IAM-Security/2-access-key.png" alt="Access Key created successfully"></p>
<hr>
<h3 id="step-3-verify-ecs-task-execution-role">Step 3: Verify ECS Task Execution Role</h3>
<p>When the Backend application runs on Amazon ECS Fargate, it needs to borrow an AWS Role to have permissions to: Pull Docker Images from ECR and write Logs to CloudWatch.</p>
<ol>
<li>On the left menu of the IAM screen, select <strong>Roles</strong>.</li>
<li>In the search box, type <code>ecsTaskExecutionRole</code>.
<ul>
<li><strong>Case 1:</strong> If this role already exists (auto-generated by AWS previously), click on it and verify it has the <code>AmazonECSTaskExecutionRolePolicy</code> policy. If it does, you are done!</li>
<li><strong>Case 2:</strong> If it does not exist, click <strong>Create role</strong>. Select Trusted entity type as <strong>AWS service</strong>, Use case as <strong>Elastic Container Service</strong> -&gt; <strong>Elastic Container Service Task</strong>. Search for and attach the <code>AmazonECSTaskExecutionRolePolicy</code> policy. Name the role <code>ecsTaskExecutionRole</code> and create it.</li>
</ul>
</li>
</ol>
<p><img src="../../images/5-Workshop/5.1-IAM-Security/3-ecs-role.png" alt="Configure ecsTaskExecutionRole"></p>
<hr>
<p>After completing the User creation and basic permissions configuration, your system is ready for the <a href="/fcj-workshop-template/5-workshop/5.2-vpc-network/">Next phase</a>.</p>
