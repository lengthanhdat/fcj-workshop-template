# Dashboard Overview & Online Examination Workflow

This section introduces the core Dashboard interface and the online examination participation experience for students on the **Aura Academic** platform.

---

### 1. Dashboard Interface

<div align="center">

![Figure 5.1: System Dashboard Interface](image.png)

**Figure 5.1. System Dashboard Interface**

</div>

**Dashboard Highlights:**
- **Overview Statistics (Stats Cards):** Quickly track key academic metrics: Average Score, Submitted Exams, Active Classes, and Practice Sessions.
- **Quick Exam Room Access:** A dedicated room code input box (e.g., `AURA25`) allows students to authenticate and join active exam rooms immediately without navigating through multiple menus.
- **Aggregated Learning Data:** Displays recent exam results, overall learning progress, and a quick summary of enrolled classes.

**Key Features:**
- **Exam Management:** Categorizes ongoing, upcoming, and completed examinations with intuitive status filtering.
- **Room Security & YOLO AI Monitoring:** A verification modal requires a valid room code. Prior to starting the exam, students enter an AI webcam verification lobby powered by YOLO Proctoring V1.0, which continuously supervises the testing environment and triggers real-time violation alerts.

**Online Testing Workflow:**
- **Optimized Workspace:** Split-screen layout separating question content (multiple-choice/essay) from a quick-navigation question palette on the right.
- **Real-Time Countdown & Auto-Submit:** Real-time countdown clock with automated answer saving and auto-submission upon time expiration to prevent data loss.
- **Immediate Grading & Summary:** Confirmation prompt before submission, followed instantly by a detailed summary of score, completion time, and correct/incorrect ratios.
